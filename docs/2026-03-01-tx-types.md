# Canonical NHBChain Transaction Types (TxType)

This document provides the definitive mapping of numeric `TxType` values required by the NHBChain L1 node. When the NHBPortal frontend (via `walletManager.ts`) constructs and signs an offline transaction, it must use the exact decimal or hex values listed below in the `type` field of the JSON payload.

> **Warning:** Previous frontend blueprint documents inferred sequential numbering (e.g., guessing POS was 12-15). Those are incorrect. Use this table as the strict source of truth.

| TxType Name                 | Decimal | Hexadecimal | Description |
|-----------------------------|---------|-------------|-------------|
| `TxTypeTransfer`            | `1`     | `0x01`      | Standard transfer of NHB |
| `TxTypeRegisterIdentity`    | `2`     | `0x02`      | Claim a username |
| `TxTypeCreateEscrow`        | `3`     | `0x03`      | Create an escrow listing |
| `TxTypeReleaseEscrow`       | `4`     | `0x04`      | Buyer releases funds to seller |
| `TxTypeRefundEscrow`        | `5`     | `0x05`      | Seller refunds funds to buyer |
| `TxTypeStake`               | `6`     | `0x06`      | Stake ZNHB |
| `TxTypeUnstake`             | `7`     | `0x07`      | Un-stake ZNHB |
| `TxTypeHeartbeat`           | `8`     | `0x08`      | Validator heartbeat for engagement |
| `TxTypeLockEscrow`          | `9`     | `0x09`      | Buyer commits to purchase |
| `TxTypeDisputeEscrow`       | `10`    | `0x0A`      | Buyer raises a dispute |
| `TxTypeArbitrateRelease`    | `11`    | `0x0B`      | Admin-only action to release |
| `TxTypeArbitrateRefund`     | `12`    | `0x0C`      | Admin-only action to refund |
| `TxTypeStakeClaim`          | `13`    | `0x0D`      | Claim matured unbonded ZNHB |
| `TxTypeMint`                | `14`    | `0x0E`      | Execute an off-chain mint voucher |
| `TxTypeSwapPayoutReceipt`   | `15`    | `0x0F`      | Record an attested swap payout |
| `TxTypeTransferZNHB`        | `16`    | `0x10`      | Standard transfer of ZNHB |
| `TxTypeSwapMint`            | `17`    | `0x11`      | Native On-Chain Swap Mint (USDT -> NHB) |
| `TxTypeSwapBurn`            | `18`    | `0x12`      | Native On-Chain Swap Burn (NHB -> USDT) |
| `TxTypePOSAuthorize`        | `32`    | `0x20`      | Pre-authorize a merchant payment |
| `TxTypePOSCapture`          | `33`    | `0x21`      | Capture an authorized payment |
| `TxTypePOSVoid`             | `34`    | `0x22`      | Void an authorized payment |
| `TxTypePOSRegistry`         | `35`    | `0x23`      | POS merchant/device registry update |

### Notes for Frontend Integration

- **POS Transactions:** Notice a numeric gap between 18 and 32. POS operations begin at `0x20`.
- **Swap Transactions:** `TxTypeSwapMint` and `TxTypeSwapBurn` MUST be used for native on-chain interactions with the Oracle instead of relying purely on REST endpoint stubs.
